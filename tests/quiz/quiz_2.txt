1. Как включить автосохранение данных в VSCODE?
    Ответ: Слева сверху есть строка меню , нужно выбрать File -> Auto Save

2. Как настроить перенос строки?
    Заходим в Preferences->Settings и в поисковой строке пишем "wrap", далее в поле Word Wrap выбираем wordWrapColumn

3. Сколько символов по pep8 разрешено на строке?
     Ответ: разрешено 79 символов

4. Какие способы переноса строк показаны в видео:
    Ответ: 2 способа:
        - Поместить код в круглые скобки и разделить в них строку на несколько строк
        - Можно разделить строку на 2 и более переменных и с помощью конкатенации совместить переменные


4.1 Строки с использованием обратного слэша (\)

string_continued = "This is a long string that we want to " \
                   "split across multiple lines."
print(string_continued)

4.2 Тройные кавычки (''' или """) 

multi_line_string = """This is a string that spans
multiple lines. You can write freely
and it will keep the line breaks."""
print(multi_line_string)

4.3 Создание списка строк и объединение с помощью join

strings = [
    "This is the first line.",
    "This is the second line.",
    "This is the third line."
]
result = "\n".join(strings)  # Используем перенос строк '\n'
print(result)

4.4 Использование круглых скобок для продолжения строки
long_string = (
    "This is a very long string that I would like to "
    "continue on the next line."
)
print(long_string)

4.5 Форматированные строки (f-строки) с использованием скобок
letter_a = 5
letter_b = 6
product_ab = letter_a * letter_b

message = (
    f"when {letter_a} is multiplied by {letter_b}, "
    f"the result is {product_ab}"
)
print(message)

4.6 Сложение строк с помощью +

string_part1 = "This is the first part, "
string_part2 = "and this is the second part."
full_string = string_part1 + string_part2
print(full_string)

5. Проверка на ошибки c помощью кнопки problems, где она находится?
     Ответ: ctrl+shift+M / cmd+shift+M (если на Мак ОС)

6. Где в vscode находится клиент гита? как в нём отправить коммит? как принять домашку?
    Ответ: Клиент гита находится в боковой панель слева. Чтобы отправить коммит , необходимо написать сообщение для коммита в поле Message, нажать на стрелочку на кнопке Commit и выбрать Commit&Push.
    Чтобы принять домашку - More Actions -> Pull.

7. Что такое GIT? он локальный? В нём можно посмотреть историю изменений файлов и вернуться к любому коммиту?
    Ответ: 
    Git — это контроль версий.
    Можно, он локальный, но его можно синхронизировать с удаленным репозиторием.
    Да, в Git можно посмотреть историю коммитов, отменить изменения и откатиться назад на нужный коммит.

8. Как вставить картинку в маркдаун? 
    Ответ: скопировать картинку в буфер, затем выбрать наш маркдаун и нажать ctrl+v / cmd+v (если на Мак ОС).

9. Где посмотреть длину строки в vs code?
    Ответ: длина строки отображена в Status Bar (Ln - номер строки, Col - длина строки), который находится снизу в vs code.

10. Как поменять тип ячейки с питона на маркдаун?
    Ответ: в правом нижнем углу ячейки есть опция Select Cell Language Mode, нажимаем на нее и меняем тип ячейки на нужный.

11. Как запустить сразу все ячейки в юпитере?
    Ответ: с помощью кнопки Run All в верхнем меню.

12. Как изменить размер картинки в юпитере? Нужно для этого знать HTML?
    Ответ: необходимо обернуть картинку в тег <img> и задать аттрибуты width и height для изменения размеров. Да, нужно знать теги в HTML.

13. Какой хоткей чтобы запустить ячейку с смещением на следующую?
    Ответ: Shift+Enter

14. Как включить отображение номеров строк в юпитере(Cell line numbers)?
    Ответ: в ячейке в правом верхнем углу нажать на три точки и затем выбрать "Show Cell Line Numbers".

15. Что такое "Go To" чем это полезно? Как перейти сразу на ошибочную ячейку?
    Ответ: Go to перебрасывает нас сразу на ячейку, в которой отвалился код при запуске.

16. Как очистить вывод ячеек которые уже запущены?
    Ответ: справа от Run All в верхнем меню есть кнопка Clear All Outputs.

17. Как работать одновременно в нескольких файлах в VSCODE? Что такое SPLIT?
    Ответ: в правом верхнем углу VS Code есть кнопка Split Edit, которая разделяет экран на 2 и более для одновременной работы сразу в нескольких файлах.
